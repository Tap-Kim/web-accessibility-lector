# 🧑‍🏫 강사용 채점 루브릭 (웹 접근성 실습)

이 문서는 입문~초급 프론트엔드 학습자 실습 결과를 빠르게 평가하기 위한 기준입니다.

## 1. 평가 원칙

- Lighthouse 점수만으로 합격/불합격을 결정하지 않습니다.
- `키보드 조작 가능 여부`와 `스크린리더 전달 품질`을 필수로 확인합니다.
- 의도된 TODO 미완료와 무관한 HTML/JS 오류로 기능이 깨진 경우 감점합니다.

## 2. 총점 구조 (100점)

| 영역 | 배점 | 핵심 기준 |
| ---- | ---- | -------- |
| 시맨틱 구조/레이블 | 25 | 랜드마크, heading, label, alt, 버튼/링크 이름 |
| 키보드 접근성 | 25 | Tab 순서, 포커스 표시, 키보드 트랩 없음 |
| 동적 상태 전달 | 25 | live region, 오류/성공/재고 상태 낭독 |
| 실무 완성도 | 25 | TODO 우선순위 준수, 코드 일관성, 회귀 오류 없음 |

## 3. 단계별 필수 통과 기준

### Step2 통과 기준

- `header/nav/main/footer` 또는 동등한 랜드마크 제공
- 모든 폼 필드에 `label` 연결
- 모든 의미 있는 이미지 `alt` 제공
- 스킵 링크로 메인 콘텐츠 이동 가능

### Step3 통과 기준

- 검색 토글의 상태 속성(`aria-expanded` 등) 동작
- 장바구니/동적 메시지 live region 전달
- 폼 검증 시 `aria-invalid`와 오류 메시지 연결
- 키보드만으로 주요 사용자 흐름(메뉴→검색→로그인) 수행 가능

### Step4 통과 기준

- 접근성 도구모음 버튼 상태(`aria-pressed`)와 UI 동기화
- 복합 위젯(검색/필터/상품 카드) 키보드 조작 가능
- 품절/재입고/긴급 상태가 스크린리더에 전달됨
- 실습 중 콘솔 치명 에러가 없어야 함

## 4. 감점 규칙

| 유형 | 감점 |
| ---- | ---- |
| 키보드로 주요 기능 1개 이상 사용 불가 | -20 |
| 폼 오류가 보조기술에 전달되지 않음 | -15 |
| 링크/버튼 접근 가능한 이름 누락(건당) | -5 |
| 의미 있는 이미지 alt 누락(건당) | -3 |
| 마크업 구조 오류로 DOM 의미 훼손 | -10 |
| 런타임 오류로 기능 중단 | -10 |

| 등급 | 점수 | 판단 기준 |
